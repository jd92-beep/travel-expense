# Weather Tab Improvement + Post-trip Archive Removal

**Date:** 2026-06-11  
**Trigger:** Weather tab loads slowly, shows too many redundant location cards, and post-trip archive in Settings is unnecessary.

---

## Root Cause Analysis

### Problem 1: Weather Tab Slow Loading
**File:** `app-compact/src/tabs/Weather.tsx:74-118`

- Component fully unmounts on tab switch (`AnimatePresence key={safeTab}` in App.tsx)
- On re-mount, `useEffect` re-runs the entire fetch pipeline
- All API calls are **sequential**: `for day → for coord → await fetchWeather()`
- 10 locations × 3-4 API calls = 30-40 sequential HTTP requests
- User sees blank/loading state every time they switch to Weather tab
- localStorage cache exists (1hr TTL) but the loop still runs to check each entry

### Problem 2: Too Many Redundant Weather Cards
**File:** `app-compact/src/lib/weather.ts:130-158` (`coordsForDay`)

- `coordsForDay(day, 6)` returns up to 6 coordinates per day
- Each spot with valid lat/lon gets its own coordinate entry
- Dedup uses `lat.toFixed(3):lon.toFixed(3)` (~110m precision) — too precise
- 10 spots in Jeju → 10 separate weather cards all showing essentially the same Jeju weather
- Card headers show individual spot names, not the grouped area name

### Problem 3: Post-trip Archive Unnecessary
**File:** `app-compact/src/tabs/Settings.tsx:626-670, 978, 1982-2017`

- `buildPostTripArchiveChecklist()` builds a checklist of 4 items (backup, share, settlement, cleanup)
- Renders as a standalone GlassCard in Settings
- All 4 action buttons just link to existing features (backup export, share preview, Stats tab, clear data)
- `toggleArchiveTrip()` at L1457 is dead code (never called)

## Phase 2: Weather Location Grouping

### Task 2.1: Add city-grouping function

**File:** `app-compact/src/lib/weather.ts` (add after `coordsForDay` at ~L158)

Add `groupedCoordsForDay(day)` that:
1. Gets all raw coordinates from `coordsForDay(day, 99)` (no limit)
2. Groups spots by proximity to known cities (from `REGION_COORDS` + `GEO_DICTIONARY`)
3. Uses Haversine distance with 30km radius threshold
4. Returns one `GroupedWeatherLocation` per city group with:
   - `label`: city name (e.g., "濟州 Jeju")
   - `lat/lon`: city centroid coordinates
   - `spotNames`: array of spot names in this group
   - `timezone`: from city or day

```typescript
export interface GroupedWeatherLocation {
  label: string;
  lat: number;
  lon: number;
  spotNames: string[];
  timezone?: string;
  missing?: boolean;
}

const GROUP_RADIUS_KM = 30;

export function groupedCoordsForDay(day: ItineraryDay): GroupedWeatherLocation[] {
  const rawCoords = coordsForDay(day, 99);
  if (rawCoords.length === 0) {
    return [{ label: weatherLocationLabel(day), lat: NaN, lon: NaN, spotNames: [], missing: true }];
  }

  // Merge REGION_COORDS + GEO_DICTIONARY as city anchors
  const cityAnchors = [
    ...Object.entries(REGION_COORDS).map(([k, v]) => ({ key: k, ...v })),
    // Add GEO_DICTIONARY entries as additional anchors
  ];

  const groups = new Map<string, { label: string; lat: number; lon: number; spots: string[] }>();

  for (const coord of rawCoords) {
    if (coord.missing) continue;
    let matched = false;
    for (const city of cityAnchors) {
      if (haversineKm(coord, city) <= GROUP_RADIUS_KM) {
        if (!groups.has(city.label)) groups.set(city.label, { label: city.label, lat: city.lat, lon: city.lon, spots: [] });
        groups.get(city.label)!.spots.push(coord.label);
        matched = true;
        break;
      }
    }
    if (!matched) {
      const key = `${coord.lat.toFixed(2)}_${coord.lon.toFixed(2)}`;
      if (!groups.has(key)) groups.set(key, { label: coord.label, lat: coord.lat, lon: coord.lon, spots: [] });
      groups.get(key)!.spots.push(coord.label);
    }
  }

  return Array.from(groups.values()).map(g => ({
    label: g.label,
    lat: g.lat,
    lon: g.lon,
    spotNames: g.spots,
  }));
}
```

## Phase 3: Weather Tab Caching & Parallel Fetch

### Task 3.1: Add module-level weather cache

**File:** `app-compact/src/lib/weather.ts` (add at top after imports)

```typescript
let _weatherRowsCache: { data: Record<string, DayWeather[]>; ts: number } | null = null;
const ROWS_CACHE_TTL = 60 * 60 * 1000;

export function getCachedWeatherRows(): Record<string, DayWeather[]> | null {
  if (_weatherRowsCache && Date.now() - _weatherRowsCache.ts < ROWS_CACHE_TTL) {
    return _weatherRowsCache.data;
  }
  return null;
}

export function setCachedWeatherRows(data: Record<string, DayWeather[]>): void {
  _weatherRowsCache = { data, ts: Date.now() };
}
```

### Task 3.2: Parallelize fetches in Weather.tsx

**File:** `app-compact/src/tabs/Weather.tsx:74-118`

Replace sequential nested loop with `Promise.all`:

```typescript
async function load() {
  const cached = getCachedWeatherRows();
  if (cached && Object.keys(cached).length > 0) {
    setRows(cached);
    setStale(true); // show subtle updating indicator
  } else {
    setBusy(true);
  }
  setError('');

  try {
    const dayPromises = displayItinerary.map(async (day) => {
      const groups = groupedCoordsForDay(day);
      const coordPromises = groups.map(async (group) => {
        if (group.missing) return { coord: group, source: '缺少座標', slots: [] };
        const { data, source } = await fetchWeather(...);
        return { coord: group, source, slots: slotsForDate(data, day.date) };
      });
      const results = await Promise.all(coordPromises);
      return { date: day.date, rows: results };
    });
    const dayResults = await Promise.all(dayPromises);
    const next: Record<string, DayWeather[]> = {};
    for (const { date, rows } of dayResults) next[date] = rows;
    setRows(next);
    setCachedWeatherRows(next);
  } catch (err) {
    setError(err instanceof Error ? err.message : String(err));
  } finally {
    setBusy(false);
    setStale(false);
  }
}
```
