## 二、推薦方案：Email 邀請 + Share Link 混合模式

### 為什麼唔用其他方案

| 方案 | 適合度 | 原因 |
|------|--------|------|
| Splitwise（群組分攤） | ❌ 過度設計 | 你個 app 係記賬，唔係分攤債務 |
| TripIt（純 Email 邀請） | ✅ 基礎好 | 但需要對方已註冊 |
| Google Docs（Email + Link） | ✅ 最佳 | 靈活，兩種方式都有 |
| WhatsApp（純 Link/Code） | ⚠️ 安全隱患 | Code 泄漏就冇辦法控制 |

### 推薦：Google Docs 風格混合模式

1. **Email 邀請**：owner 輸入對方 email → 對方收到邀請 → 登入後 accept
2. **Share Link**：owner 生成 share link/code → 任何人有 link 都可以 join
3. **角色控制**：viewer（只讀）/ editor（可以記賬）
4. **成員管理**：owner 可以移除成員、更改角色

### Phase 1: DB Migration（新 migration file）

**新建 `trip_invitations` 表**：

```sql
create table public.trip_invitations (
  id uuid primary key default gen_random_uuid(),
  trip_id uuid not null references public.trips(id) on delete cascade,
  invited_email text not null,
  invited_by uuid not null references auth.users(id),
  role text not null default 'editor'
    check (role in ('viewer', 'editor')),
  status text not null default 'pending'
    check (status in ('pending', 'accepted', 'declined', 'expired', 'revoked')),
  token text unique not null default encode(gen_random_bytes(32), 'hex'),
  expires_at timestamptz not null default now() + interval '7 days',
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
```

**新建 `trip_share_links` 表**（✅ Boss 決策：Email + Share Link 都要）：

```sql
create table public.trip_share_links (
  id uuid primary key default gen_random_uuid(),
  trip_id uuid not null references public.trips(id) on delete cascade,
  code text unique not null default upper(substring(encode(gen_random_bytes(4), 'hex') from 1 for 8)),
  role text not null default 'editor'
    check (role in ('viewer', 'editor')),
  max_uses integer,
  use_count integer not null default 0,
  expires_at timestamptz,
  revoked boolean not null default false,
  created_by uuid not null references auth.users(id),
  created_at timestamptz not null default now()
);
```

## 七、風險同注意事項

1. **Receipt Owner 放寬**：如果放寬到 editor 可以改所有人嘅 receipt，需要確認唔會破壞 Notion sync 嘅 owner scoping
2. **Storage Scope**：目前 `supabase:<user_id>` scope 可能需要調整，shared trip 嘅 localStorage state 需要特殊處理
3. **Sync Conflict**：多個 user 同時改同一張 receipt 時，用 last-write-wins + version field
4. **Credential Broker Quota**：Shared trip 嘅 AI call（scan/voice）由邊個 user quota 扣？建議由操作者自己 quota 扣
5. **Notion Mirror**：Shared trip 嘅 Notion mirror 暫時應該 disabled（每個 user 有自己嘅 Notion DB）
6. **Demo Trip**：非 Boss 用戶嘅 demo trip stripping 邏輯需要確認唔會影響 shared trips
