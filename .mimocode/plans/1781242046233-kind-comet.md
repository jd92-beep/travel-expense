## 🚨 CRITICAL (4 個)

### C1. Currency `convertAmount()` 多幣種轉換邏輯錯誤
**File**: `app-compact/src/lib/currency.ts:196-205`
**Agent**: AI/Sync

`convertAmount()` 對 JPY↔HKD 用 `jpyPerHkd(state)`（user 手動設定 rate），但對其他幣種用 snapshot rates。問題：Visa snapshot 只有 JPY rate，其他幣種會 return null，但冇 fallback 到 `FALLBACK_PER_HKD` table。用戶唔會收到任何提示，直接顯示「需要更新匯率」。

**Fix**: 當 snapshot rates 缺少某幣種時，fallback 到 `FALLBACK_PER_HKD` 常量表。

### C2. `hkd()` 函數硬編碼 JPY 而唔理 receipt 嘅 actual currency
**File**: `app-compact/src/lib/domain.ts:151-154`
**Agent**: AI/Sync

`hkd(jpy, state)` 函數名同行為都假設 input 係 JPY，但喺 `buildProps()` (notion.ts Line 343) 被用嚟計算 `hkdAmount` 作為 fallback。如果 receipt 嘅 currency 係 KRW 或 TWD，呢個 fallback 會用 JPY rate 去除，得到完全錯誤嘅 HKD 金額。**直接影響 Notion sync 嘅金額準確性。**

**Fix**: `buildProps()` 嘅 fallback 應該用 `amountToHkd(receipt.total, receipt.currency || state.tripCurrency, state)`。

### C3. AI JSON Extractor 詐騙性修補 — 錯誤 bracket 關閉導致 silent data corruption
**File**: `app-compact/src/lib/ai.ts:50-52`
**Agent**: AI/Sync

當 AI 回覆包含未關閉嘅 string 時，代碼直接 append `"` 再用 `stack.pop()` 填補缺少嘅 brackets。呢個「修補」會生成結構上有效但語義上完全錯誤嘅 JSON，例如將截斷嘅 `{"store":"ABC` 修補成 `{"store":"ABC"}`，後面所有 fields 全部丟失。

**Fix**: 喺 `endIdx === -1` 時直接 throw Error，唔好嘗試修補 broken JSON。或至少喺修補後校驗 output 嘅 field count。

### C4. Credential Broker `verifyPassword` 接受迭代次數為 1
**File**: `workers/credential-broker/src/index.js:311-312`
**Agent**: AI/Sync

`iterations < 1` 會 throw，但如果 iterations=1，PBKDF2 只跑 1 round，暴力破解成本極低。

**Fix**: 最低門檻應為 10000：`iterations < 10000`。

## ⚠️ HIGH (9 個)

### H1. `dedupeQueue` silently drops failed/error items — auto-sync 永久失效
**File**: `app-compact/src/lib/useSyncEngine.ts:36-47, 264-265`
**Agent**: Core Data

`dedupeQueue()` 過濾掉 `status === 'failed'` 或 `status === 'error'` 嘅 items。`push()` 又 skip 呢啲 items。`pendingCount()` 又 exclude 呢啲 items。結果：一個 transient network error 可以永久 disable auto-sync 而冇任何 visible error。

**Fix**: Transient failure 用 `'queued'` status（唔係 `'error'`），或者加 retry-after delay 機制。

### H2. IndexedDB `openDb()` 缺少 `onblocked` handler — concurrent opens 會 hang forever
**File**: `app-compact/src/storage/indexedDb.ts:38-48`
**Agent**: Core Data

每個 `loadIndexedState`/`saveIndexedState` 都開新 connection。如果一個 tab hold 住 DB，第二個 open 會 blocked indefinitely，Promise 永遠唔 resolve。

**Fix**: 加 `req.onblocked` handler with timeout。考慮 caching DB connection。

### H3. Dropdown menus 唔 close on outside click
**File**: `app-compact/src/tabs/Dashboard.tsx:887-933`, `Shell.tsx:339-403, 505-566`
**Agent**: UI

Trip dropdown 開咗之後冇機制關閉，user 只可以再 click trigger button。Desktop 同 mobile 都有呢個問題。

**Fix**: 加 `useEffect` with `mousedown` listener on `document` to close on outside click。

### H4. `extractJson()` brace-matching 忽略 nested strings 中嘅 escaped quotes
**File**: `app-compact/src/lib/ai.ts:26-45`, `workers/credential-broker/src/index.js:1023-1057`
**Agent**: AI/Sync

String tracking 用 `inString = !inString` toggle，但喺 string 內部遇到 `\"` 時，escape 邏輯只處理一個 escape char。JSON 允許連續 escape `\\\"` 會錯誤 toggle `inString`。

**Fix**: 用更 robust 嘅方法（regex 或 JSON parser preprocessing）。

### H5. Credential Broker AI quota — non-Supabase 用戶共享同一 quota key
**File**: `workers/credential-broker/src/index.js:522-537`
**Agent**: AI/Sync

`consumeSupabaseAiQuota` 用 `user.id` 做 hash。如果 `user` 為 `null`（session-based auth），`user?.id` 為 undefined，return null skip quota。**所有 non-Supabase 用戶可以無限使用 AI。**

**Fix**: 對 session-based 用戶，用 session token 或 IP hash 做 quota key。

### H6. Notion `pushAll()` 錯誤時繼續 push 但部分失敗無回報
**File**: `app-compact/src/lib/notion.ts:1429-1440`
**Agent**: AI/Sync

`pushAll()` 對每個 receipt 做 `pushReceipt()` 但冇 try-catch，任何一個 receipt push 失敗就會中斷整個 batch。冇收集失敗嘅 receipt IDs。

**Fix**: 加 try-catch，收集 failures，return `{ ok, failures }`。

### H7. Sync merge 時 remote receipt 會被本地 stale 嘅 `Date.now()` fallback 覆蓋
**File**: `app-compact/src/lib/syncMerge.ts:24-26`
**Agent**: AI/Sync + Core Data

本地 receipt 冇 `updatedAt`/`createdAt` 時 fallback 到 `Date.now()`（永遠最大），remote 用 `0`（永遠最小）。結果：stale local receipt 永遠 win，remote 更新被無限忽略。

**Fix**: 移除 `Date.now()` fallback，用 `createdAt` 作為唯一 fallback。

### H8. `stampReceiptForTrip()` 日期自動歸位會錯誤配對跨 trip receipts
**File**: `app-compact/src/domain/trip/normalize.ts:234-248`
**Agent**: AI/Sync

`prep-auto` 邏輯將所有日期 `<= active.endDate` 且冇 tripId 嘅 receipt 都歸入 active trip，會錯誤地將其他 trip 嘅 receipts 吸走。

**Fix**: 加日期下限檢查 `receipt.date >= active.startDate - PREP_WINDOW_DAYS`。

### H9. Credential Broker `mimoJson()` max_tokens 對 trip 太低
**File**: `workers/credential-broker/src/index.js:886-888`
**Agent**: AI/Sync

`mimoJson()` 對 `kind === 'trip'` 設 `max_tokens: 3500`，但 trip analysis prompt 要求返回完整 itinerary JSON，3500 tokens 可能唔夠，導致 JSON 截斷。

**Fix**: Trip 應該至少 8000-12000 tokens。

## 📋 MEDIUM (25 個)

### M1. `pull()` pre-computes `pending` from stale queue snapshot
**File**: `app-compact/src/lib/useSyncEngine.ts:340-422`
`pending` 在 setState 之前計算，setState callback 入面嘅 `current.syncQueue` 可能已變。`lastSyncedAt` 同 `globalSyncStatus` 可能 inconsistent。

### M2. Stale `supabaseSessionRef.current` captured inside `setState` updater closure
**File**: `app-compact/src/lib/useSyncEngine.ts:120-143`
setState updaters 可能喺 React concurrent mode 下異步執行，`supabaseSessionRef.current` 可能已變。

### M3. `isHydratingScope` false-negative on initial mount
**File**: `app-compact/src/lib/useAppState.ts:105, 217`
`hydratedScope` 同 `storageScope` 都初始化為 `'local'`，第一次 render 時 `isHydratingScope = false`，但 hydration 仲未開始。

**Fix**: 初始化 `hydratedScope` 為 `''`。

### M4. IndexedDB hydration can overwrite user edits during initial load
**File**: `app-compact/src/lib/useAppState.ts:115-124`
`stateFreshness` 比較用全局 timestamp，唔係 granular per-receipt。IndexedDB data 可能覆蓋 localStorage 入面嘅 receipt edits。

### M5. Double Supabase upsert for notion_pending receipts
**File**: `app-compact/src/lib/useSyncEngine.ts:196-204`
Notion push 前後各做一次 Supabase upsert，第二次係不必要嘅 API call。

### M6. Missing Escape key handler for modals
**File**: Multiple modal files
`ReceiptPhotoModal`、Dashboard wizard、Scan batch、Timeline editing 等所有 modals 都冇 Escape key handler。

### M7. No focus trap in any modal dialog
**File**: Multiple modal files
所有 modals 雖然有 `role="dialog"` 但冇 focus trap，Tab 可以 escape modal。

### M8. Currency toggle `<span>` 唔係 keyboard-accessible
**File**: `Dashboard.tsx:954-967`, `Stats.tsx:343-357`
`<span onClick>` 冇 `role="button"`、`tabIndex`、`onKeyDown`。Keyboard users 無法操作。

### M9. Hardcoded weather values "24°C 多雲" and "8°C"
**File**: `Dashboard.tsx:1076, 1109`
天氣顯示係 hardcoded static strings，唔係 real data。

### M10. `handleImage` stale closure over `state`
**File**: `Scan.tsx:170`
`handleImage` depends on `state`（full AppState），每次 receipt/setting update 都會 recreated。Scan 進行中嘅 state 可能 stale。

### M11. `bootSyncKeys` module-level Set persists across React StrictMode double renders
**File**: `App.tsx:42`
Module-scope `Set` 喺 StrictMode 下 effects run twice 時會 persist keys。

### M12. Dashboard wizard state not reset when closed externally
**File**: `Dashboard.tsx:1369-1372`
X button 只 reset `isWizardOpen` 同 `wizardStep`，但 form fields（name, destination, dates 等）唔 reset。Reopen 時舊 data persist。

### M13. `Settings` component 2400+ lines with no memoization
**File**: `Settings.tsx`
多個 expensive derived computations（`compactTripDoctor`、`buildSyncReadinessDryRun` 等）每次 render 都執行。

### M14. `ReceiptEditor` useEffect depends on `receipt` object reference
**File**: `ReceiptEditor.tsx:82`
如果 parent create 新 object reference for same receipt，effect 會 re-run 並 reset draft，丟失 in-progress edits。

**Fix**: 用 `receipt?.id` 作為 dependency。

### M15. `dateMs` uses UTC midnight but other date functions use local midnight
**File**: `Settings.tsx:158` vs `Dashboard.tsx:61`
`dateMs` append `Z`（UTC），`displayDateRange` 用 local time。同一日期喺唔同 timezone 會產生唔同 timestamp，可能導致 off-by-one-day errors。

### M16. Notion schema resolution — `tripIntelligence` 唔喺 resolutionOrder
**File**: `app-compact/src/lib/notion.ts:181-193`
`resolutionOrder` array 缺少 `tripIntelligence`，Notion DB 嘅 `Trip Intelligence` 欄唔會被正確解析。

### M17. Notion `pushReceipt()` photo block append 可能 append 多次
**File**: `app-compact/src/lib/notion.ts:1205-1248`
`_photoBodyBlockAdded` flag 唔喺 persisted state 中（`_` prefix），每次 retry 都會重新 append image block，導致 Notion page body 有重複圖片。

### M18. `normalizeZone()` 唔處理 SGT、PST 等常見 timezone abbreviations
**File**: `app-compact/src/domain/trip/context.ts:172-179`
只處理 JST、HKT、KST、CST。SGT 等會 fallback 到 `'auto'` 或 `'Asia/Tokyo'`。

### M19. `fileToBase64()` 正則 match 可能 return undefined groups
**File**: `app-compact/src/lib/ai.ts:164-166`
如果 `dataUrl` 唔 match，`body` 會係 `''`，return 空 base64，AI API call 會收到空 image。

### M20. `parseTextWithAi()` 當 AI return null 時會生成空 receipt
**File**: `app-compact/src/lib/ai.ts:835-836`
`Array.isArray(null)` 為 false，`rows = [null]`，會生成一個空 receipt 而唔 crash。

### M21. Credential Broker `listTrustedDevices()` 唔 filter expired devices
**File**: `workers/credential-broker/src/index.js:278-297`
過期 devices 仍然顯示喺 list 中。

### M22. `handleSaveGuideTrip` closure captures stale `state`
**File**: `App.tsx:82-138`
Async function 入面嘅 `state` 可能喺 `await upsertSupabaseTrip` resolve 時已變。

### M23. `modal-open` class toggle race condition
**File**: `Scan.tsx:88-91`, `Timeline.tsx:57`, `Settings.tsx:883`
多個組件獨立 toggle `modal-open` class on `document.documentElement`，cleanup 可能移除其他 modal 嘅 class。

### M24. `Stats` component `setState` typed as `any`
**File**: `Stats.tsx:17`
`setState: any` 同 `onTab: any` bypass TypeScript type safety。

### M25. `ymdFromText()` year fallback 用空字串
**File**: `app-compact/src/lib/ai.ts:88-97`
冇年份時 `fallback` 可能係 `''`，生成 `"-06-13"` 嘅無效日期。

**Fix**: 用 `new Date().getFullYear()` 做 fallback year。
