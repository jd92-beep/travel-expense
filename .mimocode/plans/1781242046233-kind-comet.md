## 三、Spot Extraction — 複合地名拆分

### 問題
- `道頭洞彩虹海岸道路＋石頭爺爺麥當勞` → 應該係 2 個 spots
- `午餐：On-Off / Rodem Garden / 牛島炸醬麵` → 應該係 3 個 spots
- 現有 code 完全冇 split 邏輯

### 三層 Fix 方案

#### Layer 1: AI Prompt 改善（最有效）
**File**: `app-compact/src/lib/ai.ts` (trip extraction prompt, ~L1003)

喺 Stage 2 extraction prompt 加入明確指示：
```
IMPORTANT: If a single time slot contains multiple place names separated by
'/', '＋', '+', '、', '·', '&', or '：' (after a meal/activity prefix),
you MUST create SEPARATE spot entries for EACH place, all sharing the same
time. For example:
- "午餐：On-Off / Rodem Garden / 牛島炸醬麵" → 3 spots: "On-Off", "Rodem Garden", "牛島炸醬麵"
- "道頭洞彩虹海岸道路＋石頭爺爺麥當勞" → 2 spots: "道頭洞彩虹海岸道路", "石頭爺爺麥當勞"

Each spot should have its own name, address (if known), and Google Maps link.
Strip meal/activity prefixes like "午餐：", "晚餐：", "早餐：" from place names.
```

#### Layer 2: 本地 Parser Post-processing（safety net）
**File**: `app-compact/src/lib/ai.ts` (`localSpotFromParts()` ~L365)

喺 `localSpotFromParts()` 加入 split 邏輯：
1. 檢查 name 是否包含分隔符 `/`, `＋`, `+`, `、`, `·`
2. 移除 meal/activity 前綴（`午餐：`, `晚餐：` 等）
3. Split by separators
4. 如果 split 出 >1 個 name → return array of spots instead of single spot
5. 修改 caller `extractLocalDaySpots()` 處理 array return

Splitter function：
```typescript
function splitCompoundSpotName(name: string): string[] {
  // Strip meal/activity prefix
  const stripped = name.replace(/^(早餐|午餐|晚餐|早午餐|下午茶|宵夜|brunch|lunch|dinner|breakfast)[：:]\s*/i, '');
  // Split by common separators
  const parts = stripped.split(/\s*[＋+\/、·&]\s*/).filter(Boolean);
  return parts.length > 0 ? parts : [name];
}
```

#### Layer 3: Timeline UI — 同時間多 spots 顯示
**File**: `app-compact/src/tabs/Timeline.tsx`

唔需要改 `ItinerarySpot` 結構。因為 split 後每個 spot 有自己嘅 entry（同時間），Timeline 自然會顯示為多個 cards。只需要確認：

#### Layer 4: Geo + Maps
**File**: `app-compact/src/lib/geo.ts`, `app-compact/src/lib/domain.ts`

Split 後每個 spot 獨立 geocode + 獨立 map link，現有 code 已支持，唔需要改。
